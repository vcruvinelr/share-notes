# Contributing to Share Notes

Thank you for your interest in contributing to Share Notes! This document provides guidelines and instructions for contributing.

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers
- Focus on constructive criticism
- Help others learn and grow

## How to Contribute

### Reporting Bugs

1. Check if the bug has already been reported in Issues
2. Create a new issue with:
   - Clear title and description
   - Steps to reproduce
   - Expected vs actual behavior
   - Environment details (OS, browser, versions)
   - Screenshots if applicable

### Suggesting Enhancements

1. Check if the enhancement has been suggested
2. Create a new issue with:
   - Clear title and description
   - Use case and benefits
   - Possible implementation approach

### Pull Requests

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Add/update tests
5. Update documentation
6. Submit a pull request

## Development Process

1. **Setup**
   - Follow DEVELOPMENT.md for local setup
   - Ensure all tests pass

2. **Make Changes**
   - Write clean, readable code
   - Follow code style guidelines
   - Add comments for complex logic

3. **Test**
   - Add unit tests for new features
   - Ensure all tests pass
   - Test manually in development environment

4. **Document**
   - Update README if needed
   - Add JSDoc/docstrings
   - Update ARCHITECTURE.md for architectural changes

5. **Submit**
   - Push to your fork
   - Create pull request
   - Fill out PR template
   - Link related issues

## Pull Request Guidelines

### PR Title

Use conventional commit format:
```
feat: add user profile page
fix: resolve WebSocket memory leak
docs: update deployment guide
```

### PR Description

Include:
- What changed and why
- Related issues
- Testing performed
- Screenshots (for UI changes)
- Breaking changes

### Code Review

- Respond to feedback constructively
- Make requested changes
- Keep discussion focused
- Mark conversations as resolved
